前端react项目，不管是用create-react-app 初始化的，还是用阿里的 umi 初始化项目，原则上开发语言都要用typescript，所以相关的eslint 和 prettier配置都要加入ts的相关检测特征。为了简化繁琐的配置，使用umi封装的@umijs/fabric 包对 eslint stylelint和prettier的初始配置做了封装，可初始化一些预设配置。

相关的 文档可参考

配置的步骤如下：


安装 @umijs/fabric、eslint、stylelint、prettier、typescript

1

npm i @umijs/fabric eslint stylelint typescript prettier -D

VSCODE编辑器的相关配置


检查vscode中eslint和prettier插件是否正确安装（注意，eslint一定要安装正确的版本，注意作者是Dirk Baeumer)

https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint

https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode

打开vscode命令行,输入json,打开工作区设置（只针对当前项目）

添加如下配置：

1
2
3
4
5

{
  "editor.codeActionsOnSave": {
  	"source.fixAll.eslint": true
  }
}


保存后即可生效。
